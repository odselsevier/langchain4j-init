Architecture

What Is LangChain4j?

  • LangChain4j is a Java framework for building LLM-powered apps with clean abstractions.
  • It gives you AiServices, tool-calling, chat memory, and RAG building blocks.
  • Goal: ship AI features in Java without writing glue code for every model interaction.

Think: Spring-style ergonomics for LLM workflows.

Structured Output Extraction

Declare an interface — get a strongly-typed POJO back.

public interface SupportExtractorService {

    @SystemMessage("""
        You are a support-ticket classifier. Extract the
        customer name, issue category, severity, and summary.
        """)
    TicketDetails extract(@UserMessage String rawMessage);
}
// Returned POJO (Java record)
public record TicketDetails(
    String customerName,
    String issueCategory,
    String severity,
    String summary
) {}

Zero boilerplate — no prompt templates, no JSON parsing.

Tool Calling

The LLM invokes your Java methods for real-time data.

public class InventoryTools {

    @Tool("Get current inventory count for a given SKU")
    public String checkInventory(String sku) {
        return "SKU-%s: 42 units in stock".formatted(sku);
    }
}
// Wire it in
AssistantWithTools assistant = AiServices.builder(AssistantWithTools.class)
        .chatLanguageModel(model)
        .tools(new InventoryTools())
        .build();

What Is RAG?

  • RAG = Retrieval-Augmented Generation: the model looks things up before answering.
  • Think of it as LLM + search: search finds relevant facts, the LLM explains them.
  • Result: fewer hallucinations, more grounded answers from your own docs.

Plain English: "Don’t guess from memory, read the right pages first, then answer."
