langchain4j-init

Website FAQ Demo (RAG-style retrieval + tool calling)

Java 17+ LangChain4j 1.0.0-beta3 Ollama Single Use Case

Focused on one production-like flow: website FAQ with event answers + live inventory lookup.

Problem We Solve

  • Website visitors ask mixed questions in one message: product stock + events + support details.
  • Plain LLM answers drift without retrieval context.
  • Business-critical numbers should come from tool calls, not model guesses.

Current Demo Scope

AreaStatus

Inventory wordingNow: "42 products in stock"

Old multi-use-case narrativeRemoved from this deck

Architecture

User question
  -> Main.parseQuestion(args)
  -> ToolCallingDemo.run(model, question)
  -> AssistantWithTools (AiServices proxy)
     -> WebsiteFaqTools.searchFaq(question)
     -> InventoryTools.checkInventory(sku)
  -> Assistant final natural-language response

Retrieval context and inventory values are grounded through explicit tools.

CLI Input Behavior

# default demo question
java -jar target/langchain4j-init-1.0.0-SNAPSHOT.jar

# explicit mode + custom user input
java -jar target/langchain4j-init-1.0.0-SNAPSHOT.jar faq "What events are next and stock for WIDGET-99?"

# question-only shortcut
java -jar target/langchain4j-init-1.0.0-SNAPSHOT.jar "How many products are in stock for WIDGET-99?"

Tooling Contract

InventoryTools.checkInventory(sku)
  => "SKU-WIDGET-99: 42 products in stock"

WebsiteFaqTools.searchFaq(question)
  => returns best FAQ context, including events schedule

Assistant combines both into one plain-language answer.

Example Output

INFO  ToolCallingDemo - === Website FAQ Demo (RAG-style) ===
INFO  ToolCallingDemo - User -> What upcoming events are on the website, and how many products are in stock for WIDGET-99?
INFO  ToolCallingDemo - Assistant -> Upcoming events include AI Search Webinar (2026-05-12), RAG Office Hours (2026-05-20), and Java + LLM Meetup (2026-06-02). SKU-WIDGET-99 currently has 42 products in stock.

Why This Is A Strong FAQ/RAG Use Case

  • Retrieval gives grounded event/policy context from a managed FAQ source.
  • Tool-calling returns deterministic stock data from backend logic.
  • Single response can merge both in user-friendly language for website UX.
  • Extensible: replace mock FAQ store with vector DB or CMS retrieval later.

Validation Checklist

  • User question appears in logs as User -> ....
  • Stock phrase uses products in stock everywhere.
  • Events FAQ returns concrete upcoming dates.
  • Assistant final response is plain language, not JSON/table.
